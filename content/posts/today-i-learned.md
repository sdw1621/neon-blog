# 오늘 배운 것

클로드 코드와 함께 블로그를 만들면서 웹 페이지를 이루는 세 가지 기술,
**HTML·CSS·JavaScript**가 각각 어떤 일을 하는지 알게 되었습니다. 비유하자면
집을 짓는 것과 비슷했습니다.

## HTML — 뼈대(구조)

HTML은 페이지의 **구조와 내용**을 담당합니다. 제목, 문단, 목록, 링크처럼
"무엇이 있는지"를 정의하죠. 집으로 치면 벽과 방을 나누는 **뼈대**입니다.

```html
<h1>글 제목</h1>
<p>이건 한 문단입니다.</p>
<a href="/posts/hello">글 보러 가기</a>
```

블로그의 글 목록과 본문이 화면에 글자로 나타나는 건 모두 HTML 덕분이었습니다.

## CSS — 꾸미기(디자인)

CSS는 **보이는 모습**을 담당합니다. 색, 글꼴, 여백, 정렬처럼 "어떻게 보일지"를
정하죠. 뼈대 위에 페인트를 칠하고 가구를 배치하는 **인테리어**라고 보면 됩니다.

```css
h1 {
  font-size: 2rem;
  font-weight: bold;
}
```

이 블로그에서는 Tailwind CSS를 써서 `text-2xl`, `font-bold` 같은 짧은 클래스만으로
디자인을 입혔습니다. 같은 HTML이라도 CSS에 따라 완전히 다른 느낌이 됩니다.

## JavaScript — 움직임(동작)

JavaScript는 페이지에 **동작과 상호작용**을 더합니다. 버튼 클릭, 데이터 불러오기,
화면 바꾸기 같은 "무슨 일이 일어날지"를 다루죠. 집에 들어온 **전기와 가전제품**에
가깝습니다.

```js
function greet(name) {
  return `안녕하세요, ${name}님!`;
}
```

이 블로그는 Next.js(자바스크립트 기반)가 마크다운 글을 읽어 HTML로 바꿔 주고,
페이지를 이동할 때 부드럽게 전환되도록 해 주었습니다.

## 정리

| 기술 | 역할 | 비유 |
| --- | --- | --- |
| HTML | 구조·내용 | 뼈대 |
| CSS | 디자인·모습 | 인테리어 |
| JavaScript | 동작·상호작용 | 전기·가전 |

세 가지가 각자 맡은 일을 하면서 어우러질 때 비로소 하나의 웹 페이지가 완성된다는 걸
배웠습니다. 오늘의 한 줄 정리: **HTML은 뼈대, CSS는 옷, JavaScript는 움직임.**
