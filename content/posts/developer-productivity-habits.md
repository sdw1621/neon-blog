# 개발자의 생산성을 높이는 5가지 습관

생산성은 더 빠른 노트북이나 새로운 에디터에서 나오지 않습니다. 매일 반복하는
작은 습관이 쌓여 만들어집니다. 오랜 시간 코드를 짜면서 효과를 본 다섯 가지 습관을
정리했습니다.

## 1. 하루를 '가장 어려운 일' 하나로 시작하기

아침의 집중력은 하루 중 가장 비쌉니다. 이메일이나 알림부터 확인하면 그 에너지를
잡일에 흘려보내게 됩니다. 출근 후 첫 90분은 **가장 어렵고 중요한 작업 하나**에만
씁니다. 이 한 가지를 끝내면 나머지 하루는 이미 성공한 셈입니다.

## 2. 작게 자주 커밋하기

거대한 커밋 하나는 리뷰도, 디버깅도, 되돌리기도 어렵습니다.

- 의미 있는 단위로 자주 커밋한다.
- 커밋 메시지에 "무엇을" 보다 "왜"를 적는다.
- 기능 단위 브랜치로 작업을 격리한다.

작은 커밋은 곧 작은 위험입니다. 문제가 생겨도 어디서 깨졌는지 금방 찾을 수 있습니다.

## 3. 반복 작업은 즉시 자동화하기

같은 명령을 세 번 입력했다면, 그건 스크립트가 될 신호입니다.

```bash
# 매번 치던 배포 절차를 하나의 명령으로
alias deploy="npm run build && npm run test && npm run release"
```

자동화에 쓴 10분은 앞으로 아낄 수백 번의 반복으로 돌아옵니다. 빌드, 테스트, 포매팅,
환경 세팅처럼 자주 하는 일부터 묶으세요.

## 4. 딥 워크 시간을 캘린더에 '예약'하기

집중은 의지가 아니라 환경의 문제입니다. 회의 사이에 끼어든 30분으로는 깊은 작업이
안 됩니다.

> 방해받지 않는 2시간 블록을 캘린더에 회의처럼 잡아두세요. 알림을 끄고, 한 번에
> 하나의 맥락만 다룹니다.

컨텍스트 스위칭 한 번의 비용은 생각보다 큽니다. 끊기지 않는 시간을 지키는 것이
생산성의 핵심입니다.

## 5. 하루를 5분 회고로 닫기

퇴근 전 5분, 두 가지만 적습니다.

| 질문 | 목적 |
| --- | --- |
| 오늘 무엇을 끝냈는가? | 성취를 기록하고 진행 상황을 추적 |
| 내일 가장 먼저 할 일은? | 내일 아침의 결정 비용을 제거 |

내일의 첫 작업을 미리 정해두면, 다음 날 아침 "무엇부터 하지?"로 낭비하는 시간이
사라집니다.

## 마치며

다섯 가지 모두 거창한 도구가 필요 없는 습관입니다. 한 번에 다 바꾸려 하지 말고,
이번 주에는 **하나만** 골라 시작해 보세요. 작은 습관 하나가 가장 큰 변화의 시작입니다.
